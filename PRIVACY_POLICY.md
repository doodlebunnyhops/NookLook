# Privacy Policy - NookLook Bot

**Last Updated**: December 2024

## 1. Introduction

This Privacy Policy explains how NookLook ("we", "our", "the Bot") collects, uses, and protects information when you use our Discord bot services.

## 2. Information We Collect

### 2.1 Automatically Collected Information

**Discord Platform Data:**
- Discord User ID (for command processing)
- Discord Server ID (for server-specific functionality)
- Command usage and timestamps (for analytics)
- Discord username and discriminator (for logging purposes)

**Technical Data:**
- Command response times (for performance monitoring)
- Error logs (for debugging and improvement)
- Bot interaction patterns (for feature optimization)

### 2.2 Information We Do NOT Collect

**Personal Information:**
- Real names, addresses, or contact information
- Private messages or direct messages
- Voice or video chat data
- Financial or payment information
- Location data beyond Discord server regions

**Sensitive Data:**
- No special category personal data is collected
- No biometric or health information
- No political or religious preferences

## 3. How We Use Your Information

### 3.1 Primary Uses

**Service Provision:**
- Processing and responding to Bot commands
- Providing Animal Crossing: New Horizons information
- Maintaining command history for session continuity
- Delivering personalized autocomplete suggestions

**Service Improvement:**
- Analytics to understand feature usage
- Performance monitoring and optimization
- Bug detection and resolution
- Feature development and enhancement

### 3.2 Legal Bases for Processing

- **Legitimate Interest**: Providing and improving Bot services
- **Consent**: When explicitly provided for specific features
- **Contract Performance**: Fulfilling our Terms of Service

## 4. Data Sharing and Disclosure

### 4.1 Third-Party Sharing

**We Do NOT:**
- Sell user data to third parties
- Share personal information for marketing
- Provide data to advertisers
- Transfer data for commercial purposes

**Limited Sharing:**
- Discord platform (necessary for Bot functionality)
- Error reporting services (anonymized technical data only)
- Analytics services (aggregated, non-personal data)

### 4.2 Legal Requirements

We may disclose information when legally required to:
- Comply with valid legal processes
- Protect the rights and safety of users
- Prevent fraud or abuse
- Comply with Discord's Terms of Service

### 4.3 Community and Self-Hosted Instances

**Open Source Distribution:**
- The Bot software is open source and may be hosted by community members
- Each instance operator is responsible for their own privacy compliance
- No personal data should be shared between different Bot instances
- Self-hosted instances must provide their own privacy policies

**Instance Operator Responsibilities:**
- Comply with local data protection laws (GDPR, CCPA, etc.)
- Implement appropriate security measures for user data
- Provide clear privacy policies for their specific instance
- Handle data breaches according to applicable regulations

## 5. Data Storage and Security

### 5.1 Storage Practices

**Data Location:**
- Data is stored securely in our approved hosting environment
- No personal data is stored longer than necessary
- Game data is sourced from public community databases

**Retention Periods:**
- Command logs: 30 days maximum
- Error logs: 90 days maximum
- Analytics data: 1 year maximum (aggregated only)
- No permanent personal data storage

### 5.2 Security Measures

**Technical Safeguards:**
- Encrypted data transmission
- Secure database configurations
- Regular security updates and monitoring
- Access controls and authentication

**Operational Security:**
- Limited access to data on need-to-know basis
- Regular security training for team members
- Incident response procedures
- Data backup and recovery protocols

### 5.3 Community Hosting Guidelines

**Self-Hosted Instance Security:**
- Instance operators must implement appropriate security measures
- Follow security best practices outlined in project documentation
- Regular updates to Bot software and dependencies recommended
- Monitor for security vulnerabilities and apply patches promptly

**Data Protection for Community Instances:**
- Each instance operates independently with its own data
- No centralized data sharing between community instances
- Instance operators responsible for compliance with local privacy laws
- Security guidelines provided but implementation is operator responsibility

## 6. Your Rights and Choices

### 6.1 User Rights

**Access and Control:**
- Right to know what data we have about you
- Right to request data correction or deletion
- Right to withdraw consent where applicable
- Right to data portability (where technically feasible)

**Exercising Rights:**
- Contact us through our support server
- Submit requests via GitHub issues

### 6.2 Bot Management
